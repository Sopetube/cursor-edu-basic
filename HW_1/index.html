Git Cheat Sheet

Basic command

  • git config --global user.name blabla - задати ім'я користувача (викорисутовується тільки в перший раз)
  • git config --global user.email blabal@bla - задати пошту користувача (викорисутовується тільки в перший раз)
  • git status - показує стан репозиторію(як файли відслідковуються, які зміни);
  • git init - ініціалізувати пустий репозиторій;
  • git add - перевести зміни в stage стадію;
  • git commit - комітить всі зміни і зберігає їх в історію версій (локально);
  • git log - показує історію комітів;
  • git checkout "commit id" - повертає файли і репозиторій до відповідної версії;
  • git checkout (- , master) - повернутись до останньоЇ версії;
  • git checkout "branch name" - переключається на вибрану гілку і оновлює папку до її стану;
  • git branch "branch name" - створює нову гілку;
  • git push -u - відправляє всі закомічені дані на віддалений репозиторій;
    pull = fetch + merge;
  • git merge - вносить зміни з віддаленого реп. у поточну гілку локального;
  • git clone - клонувати репозиторій з github;

Advanced command

  • git revert - відміняє останній коміт, створюючи новий який включає всі відмінені зміни;
  • git reset (--hard) - відміняє всі коміти після заданого і залишає всі зміни в робочій папці. (видаляє зміни в папці);
  • git rebase - ще один спосіб (merge) перенести зміни з однієї вітки в іншу;
  • git cherry-pick - для перенесення окремих комітів в інше місце репозиторію;
  • git stash - бере ваші зміни, готові і не готові до фіксації, зберігає їх для подальшого використання і забирає їх з робочої області;
.gitignore - в цей файл вписуємо всі файли та папки за якими не має слідкувати git та не завантажувати у віддалений репозиторій;
